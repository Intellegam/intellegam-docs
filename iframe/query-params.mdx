# Query Parameters

Configure the chat IFrame using URL query parameters. There are two types of parameters:

1. **Configuration Parameters** - Control IFrame behavior (endpoint, authentication, initial message)
2. **Context Parameters** - Pass custom data from your application (automatically forwarded to the AI backend)

## Configuration Parameters

These parameters control the IFrame's behavior and connection settings.

### endpoint

**Required** - Your AI backend API endpoint for chat requests.

```
?endpoint=https://api.example.com/chat
```

### subscriptionKey

**Optional** - API key for your backend authentication.

```
?subscriptionKey=your-api-key
```

The key is sent with chat requests to authenticate with your backend.

### query

**Optional** - Auto-send an initial message on load.

```
?query=What%20are%20the%20quarterly%20results%3F
```

The message is sent automatically when the IFrame loads.

**Note:** The query value must be URL-encoded. Spaces become `%20`, question marks become `%3F`, etc.

Example in JavaScript:
```javascript
const message = "What are the quarterly results?";
const encodedQuery = encodeURIComponent(message);
// Result: "What%20are%20the%20quarterly%20results%3F"
```

## Context Parameters

Any parameter not listed above is automatically forwarded with each chat request. Use these to pass application-specific context from your system.

### Example

```
https://ai.intellegam.com/iframe?endpoint=https://api.com/chat&customerId=123&tenantId=abc
```

The backend receives these additional parameters with each chat request:
```json
{
  "customerId": "123",
  "tenantId": "abc"
}
```

## Value Types

Parameters support various types through automatic parsing:

- **Strings**: `?name=John`
- **Numbers**: `?limit=100`
- **Booleans**: `?enabled=true`
- **Arrays**: `?tags=["urgent","important"]`
- **Objects**: `?user={"id":"123","role":"admin"}`

## Security Considerations

- Avoid sensitive data in URLs (use short-lived tokens instead)
- URL length limit: ~2000 characters
